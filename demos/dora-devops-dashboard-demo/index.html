DORA x DevOps - Interactive Demo (GitHub Pages)

This browser-only prototype accompanies the paper “Cloud-Native Resilience and DORA Compliance: A DevOps Implementation Framework”. It provides an applied perspective on how operational metrics in cloud-native systems -such as MTTR, CFR, RTO/RPO, multi-region resilience and supply-chain security- can be modelled within the regulatory framework of the Digital Operational Resilience Act (DORA).

Interactive Supplementary Material. The demo simulates DORA compliance scoring derived from DevOps practices. It allows users to adjust technical parameters, visualise results across the five DORA pillars, and export reproducible datasets in JSON format. The simulation is executed entirely in the browser, without backend dependencies.

Interpretation

  • ICT Risk: monitoring coverage, scanning, SBOM/signing, CFR, MTTR.
  • Incident Management: MTTR vs RTO alignment, alert coverage, and testing cadence.
  • Resilience Testing: chaos frequency, multi-region redundancy, immutable backups.
  • Third-Party Risk: concentration level and supply-chain controls.
  • Information Sharing: approximated through monitoring coverage and testing practices.
